GRASSMARLIN

#GRASSMARLIN

1. Product description

Discovering and cataloging SCADA and ICS host on IP-based network.

- SCADA: Supervisory Control & Data Acquisition
- ICS: Industrial Control System

(Using ~PCAP files, router and switch configuration files, CAM tables, and live network packet capture)~

Function: automatically
~determine the available networks~
~generate the network topology~
~visualize the communication between host~

~facilitate further analysis~ (by a system admin/auditor/other individual)

focus on ~organizing large sums of data to allow people to quickly make informed decisions~

1.1 Versioning Schema

Major.Minor.PatchrRevision -> Major.Minor

Major ~the set of features considered to be desirable~ (intended to be present)
eg. feature 3 - better accommodate data source while enabling greater analytic potential

Minor ~the set of feature actually present~ (maybe absent because of errors in programing, a need to gather more data before implementation, or because the developers simple need more time to produce a feature.)

Patch when ~a critical bug~ is noted in a particular Minor version, a ~Patch will be issued with the same Major and Minor version and an incremented Patch versio~ n.( don’t release Patch often)

Revision ~exact set of files in the master source repository that was used to build the released version.~
